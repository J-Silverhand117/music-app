# SOUND. — offline FLAC player (PWA)

A fully offline, installable Progressive Web App for playing a personal FLAC
library, styled after the Nothing OS design language. 100% client-side — no
backend, no network needed after the first load.

## How offline works

| What | Where it lives |
|---|---|
| App shell (HTML/CSS/JS/fonts/icons) | Service worker precache (Workbox, generated by `vite-plugin-pwa`) |
| Raw FLAC audio blobs | IndexedDB `audio` store |
| Parsed metadata (title/artist/album/track#/year/duration) | IndexedDB `tracks` store |
| Extracted cover art | IndexedDB `covers` store (one blob per album) |
| Playlists | IndexedDB `playlists` store |
| Prefs (volume, flat/vinyl view, queue, last position, shuffle/repeat) | IndexedDB `prefs` store |

Files are read **once** at import (file picker or drag-and-drop), parsed
in-browser (STREAMINFO → duration, VORBIS_COMMENT → tags, PICTURE → cover art)
and written into IndexedDB. Playback always uses object URLs created from the
stored blobs — the original files are never referenced again. The app also
calls `navigator.storage.persist()` on first import to ask the browser not to
evict the data.

## Develop

```
npm install
npm run dev
```

## Build & deploy

```
npm run build        # outputs static site into dist/
npm run preview      # serve dist/ locally at http://localhost:4173
```

`dist/` is fully static — host it anywhere (Netlify Drop, Cloudflare Pages,
GitHub Pages, a $0 static host, or your own box). `base: './'` is set, so it
works from a sub-path too.

> **Important for the phone:** service workers (and therefore offline + PWA
> install) require HTTPS or `localhost`. A plain `http://192.168.x.x` LAN URL
> will run the app but won't install or work offline. The easiest path is any
> free HTTPS static host — you only need to visit the URL once per device;
> after that everything is local.

## Install

- **Android/Chrome:** open the URL → ⋮ menu → *Add to Home screen* (or the
  install prompt). Runs standalone, black theme.
- **iOS/Safari:** Share → *Add to Home Screen*. Import uses the standard file
  picker (works with Files app); the File System Access API is deliberately
  not used.
- **Laptop (Chrome/Edge):** install icon in the address bar.

## Notes

- Regenerate the app icons with `npm run icons` (zero-dependency PNG writer in
  `scripts/gen-icons.mjs`).
- FLAC parsing is a small hand-rolled parser in `src/lib/flac.js` — no heavy
  metadata libraries.
- Albums without embedded art get a deterministic Nothing-style dot-matrix
  placeholder generated from the album name.
- Fonts: Doto (dot-matrix, stands in for Nothing's proprietary Ndot) and
  Space Grotesk, both bundled locally via Fontsource so they work offline.
- Duplicate imports are skipped (matched by file name + size).
